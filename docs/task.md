## 1. Описание задачи

В условиях работы с большим количеством текстовых заметок, Markdown-файлов или локальной документации стандартный поиск по имени файла становится неэффективным. Требуется создать легковесное консольное приложение на Python, которое позволяет индексировать содержимое текстовых файлов и выполнять по ним мгновенный полнотекстовый поиск с учетом релевантности, опечаток и подсветки найденных фрагментов.


## 2. Цель работы

Разработать прототип поискового движка на базе библиотеки Whoosh, демонстрирующий возможности индексации структурированных данных и расширенного извлечения информации без использования тяжелых внешних баз данных (Elasticsearch и др.).


## 3. Функциональные требования

Система должна обеспечивать выполнение следующих операций:

    Индексация документов: Создание (или обновление) поискового индекса на основе полей: path (уникальный ID), title, content, tags и дата изменения.

    Мультиполевой поиск: Возможность искать одновременно по заголовкам и по тексту документа.

    Ранжирование результатов: Вывод документов в порядке убывания релевантности (используя алгоритм BM25F).

    Подсветка синтаксиса (Snippets): Генерация краткого фрагмента текста из найденного документа с выделением поисковых терминов.

    Коррекция запросов: Предложение правильного написания слова («Did you mean?»), если по исходному запросу ничего не найдено.

    Удаление дубликатов: При повторной индексации файла с тем же путем (path), старая запись должна автоматически заменяться новой.

## 4. Нефункциональные требования

    Производительность: Поиск по индексу (до 1000 документов) должен занимать менее 100 мс.

    Автономность: Программа не должна требовать сторонних сервисов или Docker-контейнеров — только стандартная установка Python и библиотеки Whoosh.

    Читаемость: Код должен быть разбит на логические модули (конфигурация, движок, демонстрация).

## 5. Ограничения и допущения

    Формат данных: В рамках текущей задачи обрабатываются только строки (Unicode). Поддержка PDF или DOCX не требуется (реализуется внешними парсерами).

    Язык: Основной акцент на английский и русский языки (Whoosh поддерживает UTF-8 из коробки).

    Масштабируемость: Индекс хранится в локальной директории; многопользовательский доступ и веб-интерфейс в текущей итерации не предусмотрены.
